Software Engineering

Chapter 7: Software Engineering
Software engineering Is the branch of computer science that seeks principles to guide the development of large, complex software systems
Integrated development environment Combine tools for developing software (editors, compilers, debugging tools, etc...) into a single, integrated package
Association for Computing Machinery (ACM) An international scientific and educational organization dedicated to advancing the arts, science, and applications of information technology
Purpose of ACM serves both professional and public interests by fostering the open exchange of information and by promoting the highest professional and ethical standards.
Benefits of ACM membership Access to a large digital network of resources and skills bundle. Subscriptions to monthly magazine and discounts on other ACM offerings such as conferences a free email forwarding address, voting for ACM officers, access to CareerNews and TechNews
Student ACM membership Access to most of the above plus an additional student magazine subscription. No ability to vote for ACM officers or get the skills bundles access. No print version of the monthly magazine but access to the online version
Institute of Electrical and Electronic Engineers (IEEE) An organization of electrical, electronics, and manufacturing engineers focused on the development of standards
Purpose of IEEE IEEE's core purpose is to foster technological innovation and excellence for the benefit of humanity.
Benefits of IEEE membership Access to networking, a free professional portfolio development continuing education opportunities, discounts on services
Student IEEE membership Keeping technically current, career resources and recognition professional networking, continuing education, discounts, and humanitarian programs
Life Cycle: Development phase
Requirements analysis Goal is to specify what services the proposed system will provide to identify any conditions (time constraints, security, etc..) on those services, and to define how the outside world will interact with the system
Design Creating a plan for the construction of the propsed system
Implementation The actual writing of programs, creation of data files, and development of databases
Testing Historically the process of debugging programs and confirming that the final software product was compatible with the software requirements
Modular implementation A way of taking each unit of a prgoram and designing a solution to a programming problem using those units or modules
Unified Modeling Language (UML) A more modern collection of tools developed with the object-oriented paradigm in mind
Use case Interactions between a proposed system
Use case diagram A diagram used to depict a proposed system as a large rectangle in which interactions between the system and its users are represented as ovals and users of the system are represented as stick figures
Data Relationships
One-to-one Exemplified by the association between patients and occupied private rooms in that each patient is associated with only one room and each room is associated with only one patient
One-to-many Exemplified by the association between physicians and patients in that one physician is associated with many patients and each patient is associated with one physician
Many-to-many Would occue if we included consulting physicians in the physician-patient relationship. Then each physician could be associated with many patients and each patient could be associated with many physicians
Design pattern A predeveloped model for solving a recurring problem in software design
Software Quality Assurance (SQA) Charged with overseeing and enforcing the quality control systems adopted by the organization
Software Testing
Glass box testing This means that the software tester is aware of the interior structure of the software and uses this knowledge when designing the test
Black box testing Refers to tests that do not rely on knowledge of the software's interior composition. Performed from the user's point of view
Beta testing A preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs i real-life situations before the final version of the product is solidified and released to the market
Alpha testing Similar to beta testing but this is performed at the developer's site
Documentation
User documentation Used to explain the features of the software and describe how to use them. It is intended to be read by the user of the software and is therefore expressed in the terminology of the application
System documentation Used to describe the software's internal composition so the software can be maintained later in its life cycle
Technical documentation Used to describe how a software system should be installed and serviced (such as adjusting operating parameters, installing updates, and reporting problems back to the software's developer)
Intellectual Property Law
Software license Where the rights of the developer are formally expressed in legal terms. A legal agreement between the owner and user of a software product
Software disclaimer A statement within the software license that generally follows the following wording "In no event will Company X be liable for any damages arising out of the use of this software"
Best defenses against software liability claims Is to apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors
